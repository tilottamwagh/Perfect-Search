# Skill: Listing / Enumeration

The user wants a list or catalog of items. The query starts with "list", "show me all", "what are the", "give me", or names a category with a plural noun.

## Structure

After the TL;DR (one sentence stating how many items there are and what kind), use these sections:

### Items
Numbered list (or bulleted if order doesn't matter). Each item:
- **Bold name** as the lead
- One-sentence description
- Key attribute the user is likely scanning for (owner, date, status, version) inline
- Citation `[N]`

If the list is naturally grouped (by team, by status, by service), use sub-headings for groups.

### Filtering / sorting
If the list has more than ~10 items, briefly note what's NOT included and why — e.g., "*Filtered to open items only. 14 closed items were excluded.*"

### How to find more
One sentence pointing the user at the system or search refinement that would surface a complete list. Cite the source.

## Source quality for listings

- A **Confluence index page** or **portal listing** that's already the catalog → use it directly; this is the canonical list
- **A spreadsheet/table** in a Confluence page → high trust if recent
- **Search results aggregated by source** → fine, but note that the list reflects what surfaced in this search, not necessarily everything that exists
- **Slack mentions** → treat as anecdotal references, not a real listing

## Honesty rules

- If the search results clearly don't represent the **complete** list (e.g., search returned 30, source page suggests there are 120), say so up front: *"This list is incomplete — sources reference more items than search returned. To see the full list, check [source page]."*
- If items have **ambiguous identity** (two things with the same name), list each separately and note the disambiguator.
- Never pad the list with items from a different category just to make it longer.
